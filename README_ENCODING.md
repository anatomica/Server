# Решение проблемы с кодировкой русского текста в консоли Windows

## Проблема
При запуске сервера русский текст отображается как "РџСѓС‚СЊ" вместо "Путь".

## Решение

### Вариант 1: Использовать start.bat (рекомендуется)
Просто запустите:
```batch
start.bat
```

### Вариант 2: Использовать start.ps1 (PowerShell)
```powershell
.\start.ps1
```

### Вариант 3: Ручной запуск
Если вы запускаете напрямую через `java.exe`, **обязательно** сначала установите кодовую страницу UTF-8:

```batch
chcp 65001
java -Dfile.encoding=UTF-8 -Dconsole.encoding=UTF-8 -jar target\Server-1.0-SNAPSHOT-jar-with-dependencies.jar
```

**Важно:** Команда `chcp 65001` должна быть выполнена **в той же консоли**, где вы запускаете Java, и **перед** запуском Java.

## Почему это происходит?
Windows консоль по умолчанию использует кодовую страницу 866 (или 1251) для русского текста. Для правильного отображения UTF-8 нужно установить кодовую страницу 65001 **до** запуска Java приложения.

## Альтернативное решение
Если проблема сохраняется, можно использовать Windows Terminal (новый терминал от Microsoft), который лучше поддерживает UTF-8.
